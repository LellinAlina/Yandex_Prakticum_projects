# Принятие решений в бизнесе

## Описание проекта

Совместно с отделом маркетинга был подготовлен список гипотез для увеличения выручки. Задача проекта — приоритизировать гипотезы, провести A/B-тест и проанализировать его результаты для принятия решений.

## Цели проекта

1. Приоритизация гипотез для увеличения выручки.
2. Анализ результатов A/B-теста для принятия решений о дальнейшем развитии.

## План выполнения проекта

### Шаг 1. Приоритизация гипотез
- Оценка гипотез с использованием фреймворков **ICE** и **RICE**.
- Определение приоритетов для тестирования гипотез.

### Шаг 2. Анализ A/B-теста
1. **Предобработка данных:**
   - Проверка данных на пропуски, дубликаты и приведение типов.
   - Удаление аномалий (по перцентилям).

2. **Анализ метрик A/B-теста:**
   - Построение графиков:
     - Кумулятивной выручки по группам.
     - Кумулятивного среднего чека по группам.
     - Относительного изменения кумулятивного среднего чека группы B к группе A.
     - Кумулятивного среднего количества заказов на посетителя.
     - Относительного изменения кумулятивного среднего количества заказов.
   - Построение точечных графиков:
     - Количества заказов по пользователям.
     - Стоимости заказов.

3. **Статистический анализ:**
   - Проверка статистической значимости различий:
     - В среднем количестве заказов на посетителя между группами.
     - В среднем чеке заказа между группами.
   - Анализ "сырых" и "очищенных" данных.

4. **Принятие решения:**
   - Фиксация победы одной из групп.
   - Остановка теста при отсутствии значимых различий.
   - Рекомендация о продолжении теста.

## Описание данных

### 1. **hypothesis.csv** (Гипотезы)
- `Hypothesis` — краткое описание гипотезы.
- `Reach` — охват аудитории (по 10-балльной шкале).
- `Impact` — влияние на пользователей (по 10-балльной шкале).
- `Confidence` — уверенность в гипотезе (по 10-балльной шкале).
- `Efforts` — ресурсы на проверку гипотезы (по 10-балльной шкале).

### 2. **orders.csv** (Заказы)
- `transactionId` — идентификатор заказа.
- `visitorId` — идентификатор пользователя.
- `date` — дата заказа.
- `revenue` — сумма заказа.
- `group` — группа A/B-теста (A или B).

### 3. **visitors.csv** (Посетители)
- `date` — дата.
- `group` — группа A/B-теста.
- `visitors` — количество пользователей в указанную дату.

## Основные выводы

### Приоритизация гипотез
- Использование фреймворка **RICE** позволяет учитывать охват аудитории, что делает оценку гипотез более точной.
- Три наиболее перспективные гипотезы:
  1. Добавить форму подписки на всех основных страницах для сбора базы клиентов.
  2. Внедрить блоки рекомендаций товаров для повышения конверсии и среднего чека.
  3. Открыть два новых канала привлечения трафика для увеличения числа пользователей на 30%.
- Наименее приоритетной гипотезой остаётся изменение цвета фона главной страницы.

### Результаты A/B-теста
1. **Среднее количество заказов:**
   - Группа B показала значительное улучшение (выигрыш в 17.3%).
   - Различия статистически значимы.

2. **Средний чек:**
   - Разница между группами составила 2%, но не является статистически значимой.
   - Аномалии не повлияли на общий вывод.

3. **Решение:**
   - Остановить тест и зафиксировать победу группы B.
   - Группа B демонстрирует значительное улучшение конверсии, что подтверждено статистическим анализом.

2. Продолжить работу с гипотезами, приоритизированными по фреймворку RICE.
3. Регулярно анализировать влияние внедрённых изменений на ключевые метрики.
